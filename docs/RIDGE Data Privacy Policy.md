# RIDGE Data Privacy Policy

RIDGE collects and processes user data to provide hunting intelligence, mapping tools, community features, and safety tools.

Because hunters rely on RIDGE in remote environments and often share location-sensitive data, protecting user privacy is critical.

This document defines the **privacy architecture and policies** for the RIDGE platform.

---

# 1. Privacy Principles

RIDGE follows these privacy principles:

1. Protect user location data
2. Minimize unnecessary data collection
3. Provide user control over data sharing
4. Securely store all personal information
5. Be transparent about data usage

---

# 2. Types of Data Collected

RIDGE collects several categories of data.

## Account Data

• email address  
• username  
• profile information  
• account creation date  

---

## Location Data

RIDGE stores location information for hunting tools.

Examples include:

• stand locations  
• trail camera locations  
• hunt routes  
• map markers  

Location data is treated as **sensitive data**.

---

## Community Content

Users may upload:

• harvest photos  
• trail cam photos  
• scouting images  
• comments and posts  

This content may appear in community features like Tailgate or Trophy Room.

---

## Device Data

The system may collect:

• device type  
• operating system version  
• app version  

This information helps maintain platform performance.

---

# 3. Location Privacy Controls

Hunters often protect their hunting locations.

RIDGE provides multiple privacy levels.

Users can choose:

• exact location private  
• county level visibility  
• state level visibility  
• no location sharing

Exact GPS coordinates should **never be publicly displayed without user consent**.

---

# 4. Harvest Location Protection

For Trophy Room entries and Tailgate posts:

Location information must be limited to:

• region
• state
• optional county

Exact coordinates must remain private.

---

# 5. Data Storage

User data is stored in secure infrastructure.

Recommended storage practices include:

• encrypted databases
• secure object storage
• restricted internal access
• encrypted backups

All communication with servers must use HTTPS.

---

# 6. Data Retention

RIDGE stores user data while accounts remain active.

Users should be able to:

• delete posts
• delete harvest records
• delete accounts

Account deletion should remove or anonymize associated personal data.

---

# 7. Data Sharing

RIDGE does not sell user data.

Data may be shared only in limited cases:

• with user consent
• with legal requirements
• with trusted infrastructure providers

Any analytics data must be **aggregated and anonymized**.

---

# 8. Wildlife Data Aggregation

RIDGE may use anonymized data to generate insights about:

• wildlife movement
• hunting pressure
• habitat patterns

This information must not expose individual users or private locations.

---

# 9. Security Protections

All personal data must be protected using:

• encryption
• authentication controls
• access restrictions
• monitoring and logging

These protections are defined in `RIDGE_SECURITY_ARCHITECTURE.md`.

---

# 10. User Control

Users must be able to control:

• privacy settings
• profile visibility
• location sharing
• community participation

Privacy controls should be clearly accessible in account settings.

---

# 11. Transparency

RIDGE should clearly communicate:

• what data is collected
• how it is used
• how users can manage it

Transparency builds trust with the hunting community.

---

# 12. Privacy Responsibility

All engineers and contributors to RIDGE must respect this privacy policy.

User trust is critical for the long-term success of the platform.
